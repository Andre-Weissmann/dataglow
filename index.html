Multivariate Outliers

Approximate on-device methods. Mahalanobis distance (Mahalanobis 1936) and Isolation Forest (Liu, Ting & Zhou 2008) flag rows that are unusual across several numeric columns at once. These are heuristics, not ground truth.

SPC Control Charts & Process Capability

Shewhart 3σ control limits (Shewhart 1931) and the Cpk capability index (Six Sigma). Points outside the upper/lower control limits (UCL/LCL) signal a process out of statistical control.

Assumption Ledger

Every judgment call DATAGLOW makes while cleaning and validating — automated merges, skipped tests, gating decisions — is logged here, in plain language, so nothing is assumed silently on your behalf.

AI Synthesis On-device · optional

Optionally run a small language model entirely on your device (WebGPU) to write a plain-English synthesis of the Assumption Ledger and the 18-layer validation results. The one-time model download is generic model weights only — no dataset, row, or cell ever leaves your browser, and synthesis works fully offline afterward. This is a data-quality reasoning assistant, not a medical or diagnostic AI — it reasons only about data quality, never clinical care.

Data Provenance Trail (Chain of Custody)

A tamper-evident, SHA-256 hash-chained record of every transformation from raw file load onward. Each step's hash includes its parent's hash, so any later edit or reordering breaks verification — an audit trail for HIPAA provenance.

Peer Review Mode

Export a structured review packet — the query, key findings, all 18 layer results, and the Assumption Ledger — for a second person to review offline. They set an approve/flag decision and notes per section, then send the JSON back for you to import here. No backend, no accounts.

Time-Travel Diff
Compare a second version of your data against the active dataset

Load a second dataset file to compare against the active one. DATAGLOW diffs at three levels: row-level changes (keyed on a detected or chosen primary key), distributional drift (reusing Layer 18's fingerprint logic), and which of the 18 validation layers flip PASS↔FAIL between the two versions.

Data Time Machine

Explicitly save a snapshot of the active dataset's current state. Each snapshot records a content hash, a timestamp, and a diff summary against the previous snapshot, and is stored in your browser (IndexedDB). Click a stored snapshot to load it as the comparison dataset in the Diff view above.

Federated Fingerprinting
EXPERIMENTAL

Export a non-reversible, differentially-private statistical fingerprint of the active dataset (per-column distribution shape, coarse cardinality, and missingness — no raw values). Two sites can then compare fingerprints to see where their data's shape differs, without ever sharing row-level data. Columns with fewer than 50 rows are suppressed.

  • MAD modified z-score — Iglewicz & Hoaglin (1993)
  • IQR / Tukey fences — Tukey (1977)
  • Benford's Law — Newcomb (1881), Benford (1938)

  • Missingness framework (MCAR/MAR/MNAR) — Rubin (1976)
  • Pre-analysis checklist methodology — popularized by Atul Gawande

Red Team Mode

Loads a built-in, intentionally broken golden dataset and runs all 18 validation layers against it. Every layer must catch its target issue — that's how you know the validation system is real, not decorative.

v2: synthesize a fresh adversarial dataset that matches your currently-loaded dataset's schema — same columns and types, seeded with near-duplicate spellings, cross-column logic violations, duplicates, nulls, outliers, future dates, and schema-mismatched values — then report which seeded categories the layers catch.

Synthetic Adversarial Twin

Generate a synthetic version of your currently-loaded dataset that preserves each column's overall shape but contains no real rows. Differentially-private (Laplace) noise is added to every histogram / category count before resampling, so the synthetic data is safe to share for demos and testing.
